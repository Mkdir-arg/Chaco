# Comando /feature

Implementa una feature de forma code-first.

## Flujo

1. Entender el pedido
2. Inspeccionar el código actual
3. Diseñar el cambio si hace falta
4. Implementar con cambios mínimos
5. Validar con check/tests/revisión de templates

## Reglas

- Priorizar el código real sobre cualquier documentación histórica
- Solo generar documentación nueva si el usuario lo pide explícitamente
