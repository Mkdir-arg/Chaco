# Gobernanza del Proyecto

Esta sección define las reglas operativas para la toma de decisiones, cambios de alcance y control de calidad del proyecto Chaco.

## Objetivo

Asegurar trazabilidad, previsibilidad y acuerdos claros entre equipo funcional, desarrollo y validación.

## Roles y decisión

| Rol | Responsabilidad principal |
|---|---|
| Referente funcional | Prioriza requerimientos y aprueba entregables funcionales |
| Líder técnico | Define estrategia técnica y riesgos de implementación |
| Desarrollo | Implementa, documenta y mantiene calidad técnica |
| QA / Validación | Ejecuta pruebas, registra evidencia y reporta desvíos |

## Reglas de trabajo

1. Todo cambio relevante inicia con issue documentado.
2. Ningún desarrollo entra a sprint sin alcance y criterios de aceptación.
3. Toda entrega debe tener evidencia de validación funcional.
4. Los cambios fuera de alcance deben aprobarse explícitamente.

## Criterios mínimos de cierre

- Criterios de aceptación cumplidos o justificados.
- Evidencia de pruebas disponible.
- Documentación operativa actualizada.
- Validación funcional registrada.

## Escalamiento

1. Bloqueo funcional/técnico detectado.
2. Registro del riesgo en issue o sprint.
3. Definición de alternativa y responsable.
4. Replanificación de alcance/plazo si aplica.
