## Descripcion

Gestiona el registro, aprobación y seguimiento de instituciones de la red SEDRONAR. Una institución puede ser de 6 tipos (DTC, CAAC, CCC, CAI, IC, CT). Cada institución tiene un proceso de registro con estados (Borrador → Enviado → En Revisión → Aprobado/Rechazado), documentos requeridos y un legajo institucional con personal, evaluaciones periódicas, planes de fortalecimiento e indicadores de monitoreo.

## User Story

```
Como técnico de SEDRONAR quiero registrar y gestionar las instituciones de la red para controlar su habilitación, documentación, personal y planes de fortalecimiento institucional.
```

## Modelos principales

| Modelo | Descripcion | Campos clave |
|--------|-------------|-------------|
| `Institucion` | Institución de la red SEDRONAR | `tipo` (DTC/CAAC/CCC/CAI/IC/CT), `nombre`, `provincia`, `municipio`, `estado_registro`, `activo`, `nro_registro`, `resolucion`, `cuit`, `presta_asistencia`, `encargados` |
| `DocumentoRequerido` | Documentos del proceso de registro | `institucion`, `tipo` (20+ tipos), `archivo`, `estado`, `obligatorio` |
| `LegajoInstitucional` | Legajo de seguimiento de la institución | `institucion`, `estado_global`, `responsable_tecnico`, `fecha_ultima_evaluacion` |
| `PersonalInstitucion` | Miembros del equipo institucional | `institucion`, `nombre`, `apellido`, `rol`, `profesion`, `activo` |
| `EvaluacionInstitucional` | Evaluación periódica de la institución | `institucion`, `fecha`, `puntaje_general`, `observaciones`, `evaluador` |
| `PlanFortalecimiento` | Plan de mejora institucional | `institucion`, `objetivo`, `estado`, `configuracion_turnos` (opcional) |
| `IndicadorInstitucional` | Indicadores de monitoreo | `institucion`, `nombre`, `valor_actual`, `valor_meta`, `periodo` |

## Integraciones con otras funcionalidades

- Programas Sociales: `InstitucionPrograma` conecta instituciones con programas habilitados
- Sistema de Turnos: `Institucion` puede tener una `ConfiguracionTurnos` para ofrecer turnos
- Portal Ciudadano: las instituciones pueden auto-registrarse desde el portal
- Derivaciones: las derivaciones de legajos apuntan a instituciones como destino
- Auditoría: cambios en el estado de registro generan `AuditoriaInstitucion`

## Notas técnicas

- `DispositivoRed` y `DispositivoTipo` son alias de `Institucion` y `TipoInstitucion` para compatibilidad hacia atrás.
- El modelo tiene `unique_together = ("nombre", "municipio", "tipo")` para evitar duplicados.
- `Institucion.encargados` es ManyToMany limitado al grupo `EncargadoInstitucion`.
- El campo `configuracion_turnos` en `Institucion` (OneToOne) permite que la institución ofrezca turnos propios.
- `DocumentoRequerido` tiene 20+ tipos de documentos (personería, estatuto, habilitaciones, seguros, etc.).
