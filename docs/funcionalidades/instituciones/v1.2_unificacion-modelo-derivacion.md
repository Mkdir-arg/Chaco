# Instituciones — Unificación modelo de derivación — v1.2

> Version: v1.2
> Fecha: 2026-03-15
> Tipo: EVOLUTIVO
> App/Modulo: `legajos`

---

## Descripcion

Reemplaza el modelo `DerivacionInstitucional` por el nuevo modelo unificado `DerivacionCiudadano`, que soporta derivación desde cualquier operador (`DERIVACION`) e inscripción directa por gestores (`INSCRIPCION_DIRECTA`). La tabla legacy se conserva intacta para datos históricos. Este cambio es prerequisito directo de US-012 (derivación e inscripción de ciudadanos a programas).

## User Story

```
Como desarrollador quiero reemplazar el modelo DerivacionInstitucional
por el nuevo modelo unificado DerivacionCiudadano para eliminar la
duplicación de lógica de derivación y habilitar los flujos de US-012
```

## Criterios de aceptacion

- [x] `DerivacionCiudadano` existe con todos los campos especificados
- [x] `tipo_inicio` acepta `DERIVACION` / `INSCRIPCION_DIRECTA`
- [x] `estado` acepta solo `PENDIENTE` / `ACEPTADA` / `RECHAZADA`
- [x] `DerivacionInstitucional` sigue existiendo con related_names `*_legacy`
- [x] Data migration copia todos los registros sin pérdida; `ACEPTADA_UNIFICADA` → `ACEPTADA`
- [x] `CasoInstitucional.derivacion_origen` apunta a `DerivacionCiudadano`
- [x] `manage.py check` sin errores de `related_name`
- [x] Forms, services y views institucionales usan el nuevo modelo
- [x] Bugs preexistentes corregidos: `responsable_caso` → `responsable`, `caso.notas` → `caso.observaciones`

## Decisiones tecnicas

- `DerivacionInstitucional` no se elimina — los registros históricos se conservan con `related_name=*_legacy`. Se eliminará junto con la migración del `LegajoAtencion` a flujos en una versión futura.
- El `related_name='derivaciones'` en `InstitucionPrograma` se reasigna al nuevo modelo, por lo que `ip.derivaciones` apunta automáticamente a `DerivacionCiudadano` después de las migraciones.
- `ACEPTADA_UNIFICADA` no existe en el nuevo modelo — los casos unificados usan `ACEPTADA` con texto en `respuesta`.
- Los bugs en `CasoService` (`motivo_cierre`, `notas`) se corrigieron en el mismo slice por ser blocking para el correcto funcionamiento del servicio.

## Limitaciones conocidas / deuda tecnica

- `permissions_institucional.py`: `puede_operar_programa` tiene la lógica de `UsuarioInstitucionPrograma` comentada y retorna `False` para usuarios no-superusuario. Esto limita el uso de las vistas de aceptar/rechazar a superusuarios en producción.
- `DerivacionInstitucionalForm` sigue existiendo como dead code. Remover cuando se confirme que ninguna URL activa lo instancia.
- La UI para crear derivaciones desde el perfil del ciudadano es parte de US-012 (fuera de este alcance).
