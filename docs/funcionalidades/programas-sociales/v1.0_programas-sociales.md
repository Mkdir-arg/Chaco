## Descripcion

Gestiona el catálogo de programas sociales del sistema y permite inscribir ciudadanos en ellos, derivarlos entre programas y hacer seguimiento de su participación. El sistema de programas es dinámico: cada programa puede habilitar solapas específicas en la vista del ciudadano y configurar sus propias reglas (requiere evaluación, plan, seguimientos). También gestiona la relación programa-institución con control de cupos.

## User Story

```
Como coordinador de programas quiero inscribir ciudadanos en programas sociales, gestionar derivaciones entre programas y ver el estado de participación de cada ciudadano para hacer un seguimiento efectivo de la intervención.
```

## Modelos principales

| Modelo | Descripcion | Campos clave |
|--------|-------------|-------------|
| `Programa` | Catálogo de programas (ÑACHEC, Acompañamiento, etc.) | `codigo` (unique), `nombre`, `tipo`, `activo`, `icono`, `color`, `orden`, `requiere_evaluacion`, `requiere_plan` |
| `InscripcionPrograma` | Inscripción de un ciudadano a un programa | `ciudadano`, `programa`, `codigo` (único), `estado`, `via_ingreso`, `responsable` |
| `DerivacionPrograma` | Derivación de un ciudadano entre programas | `ciudadano`, `programa_origen`, `programa_destino`, `estado`, `urgencia`, `motivo` |
| `InstitucionPrograma` | Habilitación de institución para ejecutar programa | `institucion`, `programa`, `estado_programa`, `cupo_maximo`, `controlar_cupo` |
| `CasoInstitucional` | Caso activo de atención programática en institución | `ciudadano`, `institucion_programa`, `codigo`, `estado`, `version` |
| `DerivacionInstitucional` | Derivación de ciudadano a institución en un programa | `ciudadano`, `institucion`, `programa`, `estado`, `urgencia` |
| `CoordinadorPrograma` | Usuario que coordina un programa | `usuario`, `programa`, `activo` |

## Integraciones con otras funcionalidades

- Gestión de Ciudadanos: las inscripciones y derivaciones se hacen sobre ciudadanos
- Instituciones: `InstitucionPrograma` conecta instituciones con programas
- Legajo de Atención: los legajos pueden vincularse a programas via `InscripcionPrograma.legajo_id`
- ÑACHEC: el programa ÑACHEC tiene su propio módulo con modelos específicos (`CasoNachec`)
- Sistema de Turnos: `Programa` puede tener una `ConfiguracionTurnos` opcional
- Portal Ciudadano: los ciudadanos ven sus programas activos en `/portal/mi-perfil/programas/`

## Notas técnicas

- El modelo `Programa` es el catálogo unificado usado tanto para ciudadanos como para instituciones.
- `DerivacionPrograma.aceptar()` usa una transacción atómica para evitar condiciones de carrera al crear la inscripción.
- `InstitucionPrograma.puede_aceptar_derivaciones` valida estado global de institución + estado programático.
- El campo `Programa.modelo_legajo` permite renderizado dinámico de solapas según el tipo de programa.
