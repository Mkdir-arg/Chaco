## Descripcion

Panel principal del backoffice que presenta métricas resumidas del sistema al usuario autenticado: total de ciudadanos, legajos activos, seguimientos del día, alertas activas, usuarios activos en las últimas 24 horas y registros del mes en curso. Usa caché de 60 segundos para evitar queries costosas en cada carga.

## User Story

```
Como usuario del backoffice quiero ver un resumen del estado del sistema al iniciar sesión para tener una visión rápida de la actividad y las alertas pendientes.
```

## Notas técnicas

- El modelo `Dashboard` es un almacén clave-valor simple con un método `aumentar_cantidad()` para incrementos atómicos. No se usa directamente en la vista principal (que usa queries en tiempo real), sino como almacén auxiliar.
- Los datos de la vista se dividen en dos grupos: datos con caché (llamadas a utils) y datos dinámicos sin caché (usuarios activos en 24hs, registros del mes).
- La vista hereda de `TemplateView` con `LoginRequiredMixin`.
