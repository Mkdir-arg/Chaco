> Version: v1.0

## Descripcion

El hub del ciudadano es la vista central del backoffice desde la cual un operador accede a toda la información de un ciudadano desde un único lugar. Se accede vía `legajos:ciudadano_detalle` y utiliza un sistema de solapas (tabs) compuesto por solapas estáticas (siempre visibles) y solapas dinámicas (aparecen según los programas activos del ciudadano). Este documento registra el estado de implementación al 2026-03-19, antes de la expansión planificada en US-009.

## User Story (baseline)

```
Como operador quiero ver el hub del ciudadano con solapas organizadas para acceder a toda su información desde un solo lugar.
```

## Estado actual de solapas

### Solapas estáticas (definidas en `SolapasService.SOLAPAS_ESTATICAS`)

| ID | Nombre | Ícono | Orden | Contenido |
|----|--------|-------|-------|-----------|
| `resumen` | Resumen | dashboard | 0 | Dashboard + métricas + línea temporal + actividad reciente (AJAX) |
| `cursos_actividades` | Cursos y Actividades | school | 900 | Actividades institucionales inscriptas |
| `red_familiar` | Red Familiar | people | 998 | Contactos y vínculos familiares |
| `archivos` | Archivos | folder | 999 | Archivos adjuntos |

### Solapas dinámicas

Una solapa por cada `InscripcionPrograma` con estado `ACTIVO` o `EN_SEGUIMIENTO`. Orden `100 + programa.orden`.

| Tipo de programa | URL destino | Contenido real |
|-----------------|-------------|----------------|
| `ACOMPANAMIENTO_SOCIAL` | `legajos:programa_detalle` | Vista programática del acompañamiento |
| `NACHEC` | `nachec:detalle_caso_ciudadano` | Caso NACHEC |
| `ECONOMICO` | `programas:economico_detalle` | Placeholder (URL puede no existir) |
| `FAMILIAR` | `programas:familiar_detalle` | Placeholder (URL puede no existir) |
| (otros) | `legajos:programa_detalle` | Vista genérica |

### Badge behavior actual

- Badge = punto de color (no número)
- Solo se activa cuando `inscripcion.estado == 'PENDIENTE'` → color amarillo, texto "Pendiente"
- Para `ACTIVO` / `EN_SEGUIMIENTO` → `None` (sin badge)
- No cuenta alertas, seguimientos vencidos ni derivaciones pendientes

### Sección alertas (fuera del sistema de solapas)

Bloque independiente encima de los tabs (`#contenedor-alertas`), cargado vía AJAX desde `/legajos/ciudadanos/<id>/alertas/`. No está integrado como tab.

### Perfil social (agregado US-008, 2026-03-19)

Bloque entre header y alertas con datos habitacionales, laborales, educativos y documentación. Datos sensibles solo visibles con `puede_ver_sensible=True`.

## Limitaciones conocidas — deuda antes de US-009

1. **Solapas faltantes:** No existen tabs para Turnos, Instituciones, Conversaciones, Derivaciones, Alertas (como tab) ni Línea de tiempo (como tab independiente).
2. **Badge sin número:** Punto de color sin contador. No refleja cantidad real.
3. **Alertas fuera de solapas:** `#contenedor-alertas` está sobre los tabs — genera duplicación si se agrega tab Alertas.
4. **Legajos dentro del tab SEDRONAR:** Los `LegajoAtencion` se muestran en el tab dinámico del programa, no en un tab propio "Legajos".
5. **Línea de tiempo en Resumen:** `#timeline-container` vive dentro del tab Resumen.
6. **URLs de programas no SEDRONAR/NACHEC:** `ECONOMICO` y `FAMILIAR` apuntan a URLs que pueden no existir.
7. **Sin filtro por rol:** Todos los tabs visibles para cualquier usuario autenticado.

## Decisiones técnicas

- Solapas generadas server-side en `get_context_data`. Contenido cargado client-side vía AJAX o redirect.
- Orden controlado por campo numérico `orden` de `Programa` con offsets fijos para estáticas.
- `_normalizar_tipo_programa` incluye compatibilidad histórica con variantes de NACHEC (Ñ, codificación rota).
