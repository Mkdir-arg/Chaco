---

## Descripcion

Sistema de chat en tiempo real entre ciudadanos y operadores del sistema. Los ciudadanos pueden iniciar una conversación anónima o personal (identificándose con DNI), y los operadores la atienden desde el backoffice. Incluye sistema de cola de asignación automática con balanceo de carga, métricas de rendimiento por operador, historial de asignaciones y alertas de nuevas conversaciones.

## User Story

```
Como ciudadano quiero iniciar una consulta por chat para recibir orientación de un operador sin necesidad de asistir presencialmente. Como operador quiero gestionar mi bandeja de conversaciones, responder mensajes y ver métricas de mi desempeño.
```

## Modelos principales

| Modelo | Descripcion | Campos clave |
|--------|-------------|-------------|
| `Conversacion` | Conversación entre ciudadano y operador | `tipo` (anonima/personal), `estado` (pendiente/activa/cerrada), `prioridad`, `dni_ciudadano`, `operador_asignado`, `satisfaccion` (1-5), `tiempo_espera_segundos`, `tiempo_respuesta_segundos` |

| `MetricasOperador` | Métricas de rendimiento del operador | `operador`, `conversaciones_atendidas`, `tiempo_respuesta_promedio`, `satisfaccion_promedio` |
| `NuevaConversacionAlerta` | Alerta temporal de nueva conversación | `conversacion`, `operador`, `vista` |

## Notas técnicas

- `Conversacion.asignar_operador()` registra el historial y calcula métricas de tiempo de espera.
- `ColaAsignacion.puede_recibir_conversacion()` controla el límite de conversaciones simultáneas por operador.
- `MetricasOperador.actualizar_metricas()` usa `aggregate()` con un solo query optimizado.
- Las conversaciones pueden ser anónimas (sin identificación) o personales (con DNI).
- El campo `satisfaccion` se registra al cerrar la conversación (1 a 5).
