# Sistema de Turnos — Backoffice — v1.0

> Version: v1.0
> Fecha: 2026-03-09
> Tipo: INICIAL
> App/Modulo: `turnos`, `portal`

---

## Descripcion

Gestiona el sistema de turnos del backoffice: configuración de disponibilidades, bandeja de turnos pendientes de aprobación, agenda diaria y acciones sobre turnos (aprobar, rechazar, cancelar, completar). La arquitectura de turnos es genérica: una `ConfiguracionTurnos` puede asociarse a un Programa, una Institución, un Plan de Fortalecimiento o un RecursoTurnos. El sistema soporta modo automático (el ciudadano elige slot), manual (el operador asigna) o ambos.

## User Story

```
Como operador de backoffice quiero gestionar los turnos solicitados por ciudadanos para aprobarlos, rechazarlos o cancelarlos, y configurar la disponibilidad horaria de cada recurso o programa.
```

## Modelos principales

| Modelo | Descripcion | Campos clave |
|--------|-------------|-------------|
| `ConfiguracionTurnos` | Configuración reutilizable de turnos | `nombre`, `activo`, `requiere_aprobacion`, `modo_turno` (AUTO/MANUAL/AMBOS), `anticipacion_minima_hs`, `anticipacion_maxima_dias`, `cancelacion_hasta_hs` |
| `DisponibilidadConfiguracion` | Franja horaria semanal para una configuración | `configuracion`, `dia_semana`, `hora_inicio`, `hora_fin`, `duracion_turno_min`, `cupo_maximo` |
| `TurnoCiudadano` | Turno solicitado por un ciudadano | `ciudadano`, `configuracion`, `fecha`, `hora_inicio`, `hora_fin`, `estado` (PENDIENTE/CONFIRMADO/CANCELADO/COMPLETADO), `codigo_turno`, `motivo_consulta`, `contexto_tipo` (PROGRAMA/INSTITUCION/ACTIVIDAD/GENERICO) |
| `RecursoTurnos` | Entidad legacy que ofrece turnos | `nombre`, `tipo` (NODO/ORGANISMO/PROFESIONAL), `requiere_aprobacion` |
| `DisponibilidadTurnos` | Disponibilidad legacy para `RecursoTurnos` | `recurso`, `dia_semana`, `hora_inicio`, `hora_fin`, `duracion_turno_min`, `cupo_maximo` |

## Integraciones con otras funcionalidades

- Portal Ciudadano: el ciudadano solicita turnos desde `/portal/mi-perfil/turnos/`
- Programas Sociales: `Programa` puede tener una `ConfiguracionTurnos` (OneToOne)
- Instituciones: `Institucion` puede tener una `ConfiguracionTurnos` (OneToOne)
- Gestión de Ciudadanos: `TurnoCiudadano.ciudadano` referencia al ciudadano del sistema

## Notas técnicas

- `TurnoCiudadano.config_efectiva` resuelve la configuración efectiva con fallback al recurso legacy.
- `DisponibilidadConfiguracion.slots_preview` calcula la cantidad de slots que genera una franja horaria.
- El campo `contexto_tipo` en `TurnoCiudadano` permite distinguir si el turno es de un programa, institución, actividad o genérico, junto con `contexto_id`.
- El código de turno se genera aleatoriamente (8 caracteres alfanuméricos mayúsculos).
- Los emails de confirmación y cancelación se envían automáticamente desde `turno_aprobar` y `turno_rechazar`.
