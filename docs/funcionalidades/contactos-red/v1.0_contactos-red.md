> Version: v1.0
> Fecha: 2026-03-09
> Tipo: INICIAL
> App/Modulo: `legajos`

---

## Descripcion

Gestiona el historial de contactos entre profesionales y ciudadanos (llamadas, visitas, reuniones, mensajes), los vínculos familiares y sociales del ciudadano, los profesionales tratantes vinculados a cada legajo, los dispositivos donde el ciudadano estuvo admitido y los contactos de emergencia. También expone una línea de tiempo de todas las actividades del ciudadano y una vista de red de contactos.

## User Story

```
Como profesional de atención quiero registrar cada contacto que tengo con el ciudadano y ver la red de vínculos familiares y sociales para comprender su contexto y articular la intervención con otros actores.
```

## Modelos principales

| Modelo | Descripcion | Campos clave |
|--------|-------------|-------------|
| `HistorialContacto` | Registro de un contacto con el ciudadano | `legajo`, `tipo_contacto` (llamada/visita/reunión/etc.), `estado` (exitoso/no_contesta/etc.), `fecha_contacto`, `profesional`, `resumen`, `acuerdos`, `seguimiento_requerido`, `fecha_proximo_contacto` |
| `VinculoFamiliar` | Vínculo entre dos ciudadanos del sistema | `ciudadano_principal`, `ciudadano_vinculado`, `tipo_vinculo` (17 tipos), `es_contacto_emergencia`, `es_referente_tratamiento`, `convive` |
| `ProfesionalTratante` | Profesional asignado a un legajo | `legajo`, `usuario`, `rol` (11 roles profesionales), `es_responsable_principal`, `dispositivo`, `activo` |
| `DispositivoVinculado` | Institución donde el ciudadano estuvo admitido | `legajo`, `dispositivo`, `fecha_admision`, `fecha_egreso`, `estado` (ACTIVO/EGRESADO/DERIVADO/ABANDONO) |
| `ContactoEmergencia` | Contacto de emergencia del ciudadano | `legajo`, `nombre`, `relacion`, `telefono_principal`, `disponibilidad_24hs`, `prioridad` |

## Vistas principales

| Vista | URL | Qué hace |
|-------|-----|---------|
| `historial_contactos_simple` | `/legajos/<uuid>/historial-contactos/` | Historial de contactos de un legajo |

| `dashboard_contactos` | `/legajos/dashboard-contactos/` | Dashboard de contactos del sistema |
| `actividades_ciudadano_api` | `/legajos/ciudadanos/<id>/actividades/` | API: línea de tiempo de actividades del ciudadano |
| `timeline_ciudadano_api` | `/legajos/ciudadanos/<id>/timeline/` | API: timeline del ciudadano |
| `subir_archivos_legajo` | `/legajos/<uuid>/subir-archivos/` | Sube archivos adjuntos al legajo |
| `archivos_legajo_api` | `/legajos/<uuid>/archivos/` | API: lista de archivos del legajo |
| `archivos_ciudadano_api` | `/legajos/ciudadanos/<id>/archivos/` | API: archivos del ciudadano |
| `subir_archivos_ciudadano` | `/legajos/ciudadanos/<id>/subir-archivos/` | Sube archivos al ciudadano |
| `eliminar_archivo` | `/legajos/archivos/<id>/eliminar/` | Elimina un archivo |

## Integraciones con otras funcionalidades

- Legajo de Atención: los contactos, vínculos y profesionales tratantes están ligados al `LegajoAtencion`
- Gestión de Ciudadanos: `VinculoFamiliar` conecta ciudadanos entre sí
- Instituciones: `ProfesionalTratante.dispositivo` y `DispositivoVinculado.dispositivo` referencian instituciones

## Notas técnicas

- `ProfesionalTratante` tiene una validación que impide más de un responsable principal activo por legajo.
- `VinculoFamiliar.clean()` valida que un ciudadano no pueda vincularse consigo mismo.
- `HistorialContacto.duracion_formateada` retorna la duración en formato `Xh Ym` para llamadas y reuniones.
- El endpoint de "predicción de riesgo" (`/legajos/ciudadanos/<id>/prediccion-riesgo/`) también está en este módulo de views.
