## Descripcion

Sistema completo de auditoría y trazabilidad que registra todas las acciones de usuarios sobre datos sensibles. Tiene dos capas: auditoría general (log de acciones, descargas de archivos, sesiones de usuario, alertas de seguridad) y auditoría extendida específica por entidad (ciudadanos, legajos, evaluaciones clínicas, eventos críticos, consentimientos, derivaciones, planes de intervención, instituciones y accesos a datos sensibles). Es un requisito legal para el manejo de datos de salud.

## User Story

```
Como administrador del sistema quiero acceder al historial completo de cambios en datos sensibles para auditar quién modificó qué, cuándo y desde dónde, y detectar actividad sospechosa.
```

## Modelos principales

| Modelo | Descripcion | Campos clave |
|--------|-------------|-------------|
| `LogAccion` | Log genérico de todas las acciones | `usuario`, `accion` (LOGIN/LOGOUT/CREATE/UPDATE/DELETE/VIEW/DOWNLOAD/EXPORT), `modelo`, `objeto_id`, `ip_address`, `timestamp` |
| `LogDescargaArchivo` | Log específico de descargas | `usuario`, `archivo_nombre`, `archivo_path`, `ip_address`, `timestamp` |
| `SesionUsuario` | Tracking de sesiones activas/cerradas | `usuario`, `session_key`, `ip_address`, `inicio_sesion`, `ultima_actividad`, `activa` |
| `AlertaAuditoria` | Alertas automáticas de seguridad | `tipo` (MULTIPLES_LOGINS/ACTIVIDAD_SOSPECHOSA/DESCARGA_MASIVA/ACCESO_FUERA_HORARIO/CAMBIOS_CRITICOS), `severidad`, `revisada` |
| `AuditoriaCiudadano` | Auditoría de cambios en ciudadanos | `ciudadano`, `accion`, `campos_modificados`, `datos_anteriores`, `datos_nuevos`, `modifico_datos_personales` |
| `AuditoriaLegajo` | Auditoría de cambios en legajos | `legajo`, `campo_modificado`, `valor_anterior`, `valor_nuevo`, `cambio_estado`, `cambio_responsable`, `cambio_nivel_riesgo` |
| `AuditoriaEvaluacion` | Auditoría de evaluaciones clínicas | `evaluacion`, `genera_alerta`, `cambio_riesgo_suicida`, `cambio_violencia`, `riesgo_suicida_anterior`, `riesgo_suicida_nuevo` |
| `AuditoriaEventoCritico` | Auditoría de eventos críticos | `evento`, `tipo_evento`, `notificados` |
| `AuditoriaConsentimiento` | Auditoría de consentimientos (req. legal) | `consentimiento`, `datos_completos`, `archivo_hash` (SHA256), `aprobado_por` |
| `AuditoriaAccesoSensible` | Acceso a campos sensibles | `content_type`, `object_id`, `tipo_acceso`, `campos_accedidos`, `fuera_horario`, `acceso_multiple` |
| `AuditoriaDerivacion` | Auditoría de derivaciones | `derivacion`, `estado_anterior`, `estado_nuevo`, `institucion_origen`, `institucion_destino` |
| `AuditoriaPlanIntervencion` | Auditoría de planes de intervención | `plan`, `cambio_vigencia`, `vigente_anterior`, `vigente_nuevo` |
| `AuditoriaInstitucion` | Auditoría de cambios en instituciones | `institucion`, `cambio_estado_registro`, `cambio_activo` |
| `OperacionMasiva` | Registro de imports/exports/cambios batch | `tipo`, `modelo_afectado`, `cantidad_registros`, `registros_exitosos`, `registros_fallidos`, `archivo_origen` |

## Integraciones con otras funcionalidades

- Todas las funcionalidades: cada módulo del sistema escribe en los modelos de auditoría correspondientes
- Usuarios y Permisos: `LogAccion` y `SesionUsuario` tracean la actividad de cada usuario
- Legajo de Atención: `AuditoriaLegajo` y `AuditoriaEvaluacion` tracean cambios clínicos
- Instituciones: `AuditoriaInstitucion` tracean cambios de estado de registro

## Notas técnicas

- `AuditoriaEvaluacion.genera_alerta` se pone en `True` cuando cambia `riesgo_suicida` o `violencia`, disparando alertas de alta prioridad.
- `AuditoriaConsentimiento` requiere justificación obligatoria para acciones DELETE y aprobación de un supervisor.
- `AuditoriaAccesoSensible` usa `GenericForeignKey` para poder referenciar cualquier modelo del sistema.
- `OperacionMasiva` registra imports/exports masivos con conteo de exitosos/fallidos y archivo de log.
- El acceso al dashboard de auditoría requiere ser `superuser` o pertenecer al grupo `Administrador`.
