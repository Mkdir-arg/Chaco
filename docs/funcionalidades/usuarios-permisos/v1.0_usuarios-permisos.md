## Descripcion

Gestiona los usuarios del backoffice (staff) del sistema: login, CRUD de usuarios, asignación a grupos de permisos y cambio de email con verificación por token. El sistema de permisos se basa en los grupos de Django (`auth.Group`) con nombres descriptivos como "Usuario Ver", "Usuario Crear", etc. Los usuarios del portal ciudadano son un caso aparte gestionado desde el módulo `portal`.

## User Story

```
Como administrador del sistema quiero gestionar los usuarios del backoffice, asignarles grupos de permisos y controlar el acceso a cada funcionalidad para garantizar la seguridad y trazabilidad del sistema.
```

## Modelos principales

| Modelo | Descripcion | Campos clave |
|--------|-------------|-------------|
| `Profile` | Perfil extendido del usuario (OneToOne con `User`) | `user`, `dark_mode`, `es_usuario_provincial`, `provincia`, `rol` |
| `SolicitudCambioEmail` | Solicitud de cambio de email con confirmación por token | `user`, `nuevo_email`, `token` (UUID), `confirmado`, `expirado` |

| Vista | URL | Qué hace |
|-------|-----|---------|
| `UsuariosLoginView` | `/` (raíz) | Login del backoffice |

| `UserListView` | `/usuarios/` | Lista de usuarios (requiere grupo "Usuario Ver") |

| `UserDeleteView` | `/usuarios/eliminar/<pk>/` | Eliminación de usuario (requiere grupo "Usuario Eliminar") |
| `GroupListView` | `/grupos/` | Lista de grupos de permisos (requiere grupo "Grupos Ver") |

## Integraciones con otras funcionalidades

- Auditoría: `LogAccion` y `SesionUsuario` registran la actividad de cada usuario
- Portal Ciudadano: los ciudadanos tienen su propio sistema de login separado (`portal/views_ciudadano.py`)
- Legajos: los usuarios son los `responsable` de los legajos
- Todos los módulos: el sistema de grupos controla el acceso a cada funcionalidad

## Notas técnicas

- El acceso a cada vista está controlado por el decorador `group_required(['Nombre del Grupo'])` definido en `core/decorators.py`.
- Los grupos de permisos siguen un patrón: `[Entidad] [Accion]` (ej: "Usuario Ver", "Legajo Crear").
- `Profile` extiende el `User` de Django con un OneToOne; se crea automáticamente vía señal.
- `SolicitudCambioEmail.esta_vigente` verifica que el token no haya expirado (24 horas).
- El campo `es_usuario_provincial` en `Profile` permite filtrar vistas por provincia para usuarios regionales.
- El grupo `EncargadoInstitucion` tiene un rol especial: limita el `M2M` de encargados en `Institucion`.
