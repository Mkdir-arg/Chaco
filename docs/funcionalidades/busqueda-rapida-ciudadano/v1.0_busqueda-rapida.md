## Descripcion

Widget de búsqueda en tiempo real integrado en la navbar del backoffice. Permite buscar ciudadanos por DNI exacto o por nombre/apellido (mínimo 3 caracteres) desde cualquier pantalla del sistema, con resultados desplegables que muestran nombre, DNI y edad. Diseñado para atender consultas telefónicas sin navegar al módulo de ciudadanos.

## User Story

```
Como operador quiero poder buscar un ciudadano por nombre o DNI de forma rápida para atender consultas telefónicas sin demoras
```

## Criterios de aceptacion

- [x] Si se ingresan solo dígitos, busca por DNI exacto
- [x] Si se ingresan 3+ caracteres de texto, busca por nombre o apellido (icontains)
- [x] El dropdown aparece con debounce de 300ms para no saturar el servidor
- [x] Se muestran máximo 10 resultados con nombre completo, DNI y edad calculada
- [x] Solo acceden usuarios con rol `ciudadanoVer` o `ciudadanoCrear`
- [x] La búsqueda se integra en la navbar sin romper el layout existente

## Dependencias

- Depende de: grupos Django `ciudadanoVer`, `ciudadanoCrear` (creados por US-011)
- Usada por: backoffice (todos los módulos vía navbar)

## Decisiones tecnicas

- **Alpine.js + fetch nativo**: sin librerías adicionales, consistente con el resto del sistema.
- **Debounce en el cliente**: 300ms para no lanzar request en cada tecla.
- **Avatar con inicial**: se muestra la primera letra del apellido en lugar de foto para evitar campo adicional en el modelo y la complejidad de la URL de foto.
- **No condicional de template**: la navbar no filtra por grupo con template tags (no funciona con `.filter()`). El API devuelve `[]` si el usuario no tiene permiso — sin errores visibles.
- **Posición de URL**: la ruta `ciudadanos/buscar/` se registra antes de `ciudadanos/<int:pk>/` para evitar que "buscar" sea interpretado como un pk entero.

## Limitaciones conocidas / deuda tecnica

- La búsqueda no incluye ciudadanos inactivos (intencional, pero podría necesitar un toggle de "incluir inactivos").
- `foto_url` siempre devuelve `null` — si en el futuro se agrega foto al modelo, se puede completar sin breaking change.
- No hay paginación: máximo 10 resultados. Si se necesitan más, se debe ir al módulo de ciudadanos.
