## Descripcion

Agrega la gestión de clases individuales dentro de una actividad institucional y el registro de asistencia por clase. Los operadores pueden crear, editar y eliminar clases, y registrar la asistencia (PRESENTE / AUSENTE / JUSTIFICADO / TARDANZA) para cada ciudadano inscripto. Los ciudadanos pueden consultar su historial de asistencia desde el portal.

## User Story

```
Como operador institucional quiero registrar clases dentro de una actividad
y marcar la asistencia de cada ciudadano inscripto en cada clase para llevar
un control preciso de participación por sesión.

Como ciudadano quiero ver mi historial de asistencia en las actividades en las
que estoy inscripto para conocer mi porcentaje de participación.
```

## Criterios de aceptacion

- [x] Un operador puede crear una clase con fecha, hora de inicio, duración (opcional) y título (opcional)
- [x] No pueden existir dos clases en la misma actividad con igual fecha y hora (unicidad con UniqueConstraint)
- [x] La lista de clases muestra presentes/total y badge "Futura" para clases con fecha > hoy
- [x] La asistencia solo es editable en clases pasadas o de hoy; las futuras muestran un aviso de solo lectura
- [x] La nómina de asistencia incluye solo inscriptos INSCRITO/ACTIVO cuya fecha de inscripción es anterior o igual a la clase
- [x] El estado por defecto al tomar asistencia es AUSENTE; existe PRESENTE, TARDANZA y JUSTIFICADO
- [x] Eliminar una clase con asistencias registradas muestra error (no se puede)
- [x] El ciudadano puede ver su historial de asistencia por actividad con porcentaje calculado
- [x] El porcentaje cuenta PRESENTE + TARDANZA sobre total de clases pasadas

## Decisiones tecnicas

- `AsistenciaClase.inscripcion` FK a `InscriptoActividad` (no a `Ciudadano`) para vincular la asistencia a la inscripción concreta, facilitando controles por código y estado.
- `related_name='asistencias_clase'` en `AsistenciaClase.inscripcion` para no colisionar con `related_name='asistencias'` del modelo legacy `RegistroAsistencia`.
- `ClaseActividad.es_futura` como `@property` (no campo calculado en DB) — suficiente para la UI, evita query extra.
- Nómina filtrada por `estado IN ['INSCRITO','ACTIVO']` y `creado__date <= clase.fecha` para respetar inscripciones posteriores a la clase.
- `UniqueConstraint` (no `unique_together` deprecado) para `(actividad, fecha, hora_inicio)` y `(clase, inscripcion)`.
- Servicio `guardar_asistencia_clase` usa `upsert` lógico con `update_or_create` para idempotencia.
- `colorMap` JS definido en `{% block extra_js %}` del template de asistencia para colorear selects dinámicamente sin jQuery.

## Limitaciones conocidas / deuda tecnica

- No hay exportación de asistencia a Excel/PDF.
- No hay vista de resumen de asistencia global por actividad (porcentaje de todos los inscriptos de una vez).
- El campo `observaciones` en `AsistenciaClase` existe en el modelo pero no se expone en la UI actual.

## Cambios respecto a version anterior

v1.1 incorporó inscripción de ciudadanos. v1.2 agrega la capa de clases (sesiones) por encima de la inscripción y permite registrar asistencia clase por clase. El modelo legacy `RegistroAsistencia` (por día, sin granularidad de clase) se mantiene sin modificar.
