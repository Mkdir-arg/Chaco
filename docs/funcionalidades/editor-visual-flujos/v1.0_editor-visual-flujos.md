## Descripcion

Editor visual drag & drop para diseñar flujos de atención de programas sociales. Permite a usuarios con permiso `programaConfigurar` construir grafos de nodos y transiciones sin editar JSON manualmente. Consume la API REST del motor de flujos (US-006) para guardar borradores y publicar versiones. Implementado con React Flow + Vite, embebido en una página Django del backoffice.

## User Story

```
Como usuario con rol programaConfigurar
quiero un editor visual drag & drop para diseñar el flujo de un programa
para poder configurar visualmente la secuencia de pasos que guía la atención
de un ciudadano desde su inscripción hasta el cierre del caso,
sin necesidad de editar JSON manualmente
```

## Criterios de aceptacion

- [x] `GET /flujos/programas/<id>/flujo/editar/` — accesible solo con permiso `programaConfigurar`
- [x] Al cargar, hace GET a la API y renderiza nodos y transiciones existentes; si no hay flujo, canvas con nodo Inicio por defecto
- [x] Panel izquierdo con 5 tipos de nodo: `inicio`, `fin`, `accion_humana`, `espera`, `decision`
- [x] Solo un nodo `inicio` permitido — tipo deshabilitado si ya existe uno en el canvas
- [x] Drag & drop de nodos al canvas; eliminación con tecla Delete
- [x] Conexión entre nodos arrastrando puerto de salida al de entrada; eliminación con click
- [x] Panel derecho de propiedades: nombre y descripción por nodo; condición editable en transiciones de nodos `decision`
- [x] Condición configurable: campo, operador (`==` `!=` `>` `>=` `<` `<=` `in`), valor; o libre (null)
- [x] Validación en tiempo real: errores bloqueantes (sin inicio/fin) y advertencias (nodos sueltos, sin nombre)
- [x] "Guardar borrador" — siempre habilitado, llama POST /api/flujos/<id>/definicion/
- [x] "Publicar flujo" — habilitado solo si pasa validaciones; guarda borrador y luego publica
- [x] Toast de confirmación/error en ambas acciones; manejo de sesión expirada (403)
- [x] Aviso informativo al cargar si hay instancias activas en la versión publicada
- [x] URLs de API inyectadas desde Django via `window.flowEditorConfig` — no hardcodeadas en el bundle
- [x] Enlace al editor desde el listado de programas (ícono `fa-project-diagram`)

## Decisiones tecnicas

- **React embebido en Django**: el bundle React se sirve desde `static/flujos/dist/` via Django staticfiles. El template renderiza el mount point y pasa configuración via `window.flowEditorConfig`. No es una SPA con routing propio — sigue DT-004.
- **Namespace separado `flujos_editor`**: el namespace `flujos` ya está ocupado por la API. La URL del editor usa el namespace `flujos_editor` registrado con un include separado (`flujos/views_urls.py`) bajo el prefijo `/flujos/`. Evita colisión sin mover URLs.
- **Vite con nombres de archivo fijos**: `rollupOptions.output` fija `entryFileNames: 'assets/index.js'` y `assetFileNames: 'assets/[name].[ext]'` sin hash. El template Django puede referenciar los assets con paths estáticos predecibles.
- **CSRF desde cookie**: el editor React lee el token desde `document.cookie` (`csrftoken`) e incluye `X-CSRFToken` en todos los POST. Django no tiene `CSRF_COOKIE_HTTPONLY=True` (default False), así que la cookie es legible desde JS.
- **posición de nodos en `config`**: la posición `{x, y}` de cada nodo en el canvas se guarda en el campo `config` del JSON de definición para que al recargar el editor los nodos aparezcan donde el usuario los dejó.

## Limitaciones conocidas / deuda tecnica

- El bundle debe compilarse manualmente con `cd frontend/flow-editor && npm install && npm run build` antes de `collectstatic`. No hay paso automatizado en CI/CD todavía.
- El editor tiene el sidebar del backoffice presente, reduciendo el espacio del canvas. Pantalla completa (sin sidebar) es deuda de UX futura.
- Los tipos de nodo extendidos del requerimiento original (Formulario, Email, Aprobación, Tarea territorial) requieren primero extender el runtime backend — deuda post US-006.
- La validación de grafo totalmente conexo (alcanzabilidad desde el nodo Inicio) no está implementada — solo se validan nodos completamente desconectados. Deuda futura.
- El historial de versiones navegable (ver/comparar versiones anteriores) no está implementado. Deuda futura.
