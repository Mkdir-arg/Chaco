## Descripcion

Sistema de alertas automáticas que notifica a los profesionales cuando ocurren eventos que requieren atención inmediata. Existen dos tipos: alertas generadas por eventos críticos en legajos (sobredosis, recaída, etc.) y alertas generales sobre ciudadanos. Las alertas tienen un dashboard propio y se muestran en tiempo real mediante endpoints AJAX.

## User Story

```
Como profesional de atención quiero recibir alertas inmediatas cuando ocurra un evento crítico en un legajo bajo mi responsabilidad para poder actuar a tiempo ante situaciones de riesgo.
```

## Modelos principales

| Modelo | Descripcion | Campos clave |
|--------|-------------|-------------|
| `AlertaEventoCritico` | Alerta generada por un evento crítico | `evento` (FK a `EventoCritico`), `destinatarios` (M2M a User), `leida`, `fecha_lectura` |
| `AlertaCiudadano` | Alerta general sobre un ciudadano | `ciudadano`, `tipo`, `mensaje`, `resuelta`, `fecha_resolucion`, `resuelta_por` |

## Vistas principales

| Vista | URL | Qué hace |
|-------|-----|---------|
| `alertas_dashboard` | `/legajos/alertas/` | Dashboard de todas las alertas activas |
| `cerrar_alerta_ajax` | `/legajos/alertas/<id>/cerrar-ajax/` | Marca una alerta como resuelta (AJAX) |
| `alertas_count_ajax` | `/legajos/alertas/count/` | Retorna el conteo de alertas no leídas (AJAX) |
| `alertas_preview_ajax` | `/legajos/alertas/preview/` | Preview de alertas para el navbar (AJAX) |
| `alertas_ciudadano_api` | `/legajos/ciudadanos/<id>/alertas/` | Alertas de un ciudadano específico (API) |
| `cerrar_alerta_api` | `/legajos/alertas/<id>/cerrar/` | Cierra una alerta de ciudadano (API) |
| `CerrarAlertaEventoView` | `/legajos/cerrar-alerta/` | Cierra una alerta de evento crítico |

## Integraciones con otras funcionalidades

- Legajo de Atención: las alertas se generan automáticamente cuando se crea un `EventoCritico`
- Gestión de Ciudadanos: `AlertaCiudadano` está vinculada directamente al ciudadano
- Dashboard: el dashboard principal muestra el conteo de alertas activas via `contar_alertas_activas()`
- Auditoría: los eventos críticos que generan alertas tienen su propia tabla `AuditoriaEventoCritico`

## Notas técnicas

- Las alertas se crean automáticamente desde la vista de `EventoCriticoCreateView` al guardar el evento.
- El conteo de alertas se usa en el navbar para mostrar una notificación al usuario logueado.
- `AlertaCiudadano` es independiente de `EventoCritico` y puede generarse desde cualquier parte del sistema.
- Existe también `AlertaAuditoria` en `core/models_auditoria.py` para alertas de seguridad (múltiples logins, accesos sospechosos), que es un sistema separado.
