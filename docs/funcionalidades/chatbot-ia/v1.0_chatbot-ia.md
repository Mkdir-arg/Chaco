## Descripcion

Asistente de inteligencia artificial integrado al backoffice del sistema. Permite a los usuarios internos (staff) mantener conversaciones con un modelo de lenguaje (LLM) para obtener asistencia, consultar documentación o resolver dudas operativas. Incluye base de conocimiento configurable, sistema de feedback por mensaje y panel de administración para gestionar la API key y los logs.

## User Story

```
Como usuario del backoffice quiero tener acceso a un asistente de IA para resolver dudas operativas, consultar procedimientos y obtener asistencia sin salir del sistema.
```

## Modelos principales

| Modelo | Descripcion | Campos clave |
|--------|-------------|-------------|
| `Conversation` | Sesión de chat con el bot | `user`, `title`, `is_active` |
| `Message` | Mensaje individual dentro de la sesión | `conversation`, `role` (user/assistant), `content`, `tokens_used` |
| `ChatbotKnowledge` | Base de conocimiento configurable | `title`, `content`, `category`, `is_active` |
| `ChatbotFeedback` | Feedback del usuario sobre una respuesta | `message`, `user`, `rating` (1-5), `comment` |

| Vista | URL | Qué hace |

| `chat_interface` | `/chatbot/` | Interfaz principal del chatbot |

| `submit_feedback` | `/chatbot/feedback/` | Registra feedback sobre una respuesta |
| `admin_data` | `/chatbot/api/admin-data/` | Estadísticas para el panel admin |
| `chat_logs` | `/chatbot/api/logs/` | Logs de conversaciones (admin) |

## Notas técnicas

- El chatbot usa un LLM externo (la API key se configura desde el panel de admin, no desde settings).
- `ChatbotKnowledge` actúa como contexto adicional inyectado en los prompts.
- El conteo de tokens se registra en cada `Message` para control de costos.
- Este módulo es independiente del sistema de `Conversacion` (chat ciudadano-operador), que es un producto diferente.
