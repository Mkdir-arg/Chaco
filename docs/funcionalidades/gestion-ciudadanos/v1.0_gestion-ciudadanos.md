## Descripcion

Permite registrar, buscar, ver y editar ciudadanos dentro del sistema. Un ciudadano es la entidad central del sistema: toda atención, legajo, programa y turno se articula alrededor de él. El registro puede iniciarse via confirmación RENAPER (con DNI + sexo) o de forma manual cuando el ciudadano no se puede validar por API.

## User Story

```
Como operador del sistema quiero registrar y gestionar ciudadanos para poder abrirles legajos, inscribirlos en programas y hacer seguimiento de su situación.
```

## Modelos principales

| Modelo | Descripcion | Campos clave |
|--------|-------------|-------------|
| `Ciudadano` | Persona registrada en el sistema | `dni` (unique), `nombre`, `apellido`, `fecha_nacimiento`, `genero`, `telefono`, `email`, `domicilio`, `provincia`, `municipio` |

## Vistas principales

| Vista | URL | Qué hace |
|-------|-----|---------|
| `CiudadanoListView` | `/legajos/ciudadanos/` | Lista de ciudadanos con búsqueda |
| `CiudadanoCreateView` | `/legajos/ciudadanos/nuevo/` | Inicia el flujo de alta (redirige a confirmar o manual) |
| `CiudadanoConfirmarView` | `/legajos/ciudadanos/confirmar/` | Confirma datos del ciudadano obtenidos de RENAPER |
| `CiudadanoManualView` | `/legajos/ciudadanos/manual/` | Alta manual sin validación RENAPER |
| `CiudadanoDetailView` | `/legajos/ciudadanos/<pk>/` | Perfil completo del ciudadano (con solapas: legajos, programas, alertas, contactos, etc.) |
| `CiudadanoUpdateView` | `/legajos/ciudadanos/<pk>/editar/` | Edición de datos del ciudadano |

## Integraciones con otras funcionalidades

- Confirmación RENAPER: el alta normal pasa por verificación via API RENAPER (ver doc `confirmar-ciudadano-renaper/`)
- Legajo de Atención: un ciudadano puede tener múltiples legajos
- Programas Sociales: un ciudadano puede estar inscripto en múltiples programas
- Sistema de Turnos: los turnos del portal están vinculados al ciudadano
- Conversaciones: las conversaciones de chat pueden asociarse al DNI del ciudadano
- Alertas: `AlertaCiudadano` se genera automáticamente ante eventos críticos

## Notas técnicas

- El campo `dni` es único en la base de datos y actúa como identificador natural.
- El modelo hereda de `TimeStamped` (base abstracta con `creado`/`modificado`).
- La vista de detalle del ciudadano actúa como hub central con solapas dinámicas que muestran legajos, programas, contactos, alertas y línea de tiempo.
- La predicción de riesgo tiene un endpoint API en `/legajos/ciudadanos/<id>/prediccion-riesgo/`.
