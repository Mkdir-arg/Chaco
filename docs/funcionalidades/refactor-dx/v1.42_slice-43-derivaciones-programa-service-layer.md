# Refactor DX Interno — Slice 43

> Fecha: 2026-03-13
> Estado: Implementado
> Apps afectadas: `legajos`

## User Story

Como equipo de desarrollo quiero sacar la lógica sensible de `views_derivacion_programa.py` a un service antes de modularizar físicamente ese borde con `ÑACHEC`.

## Alcance

- Crear `legajos/services/derivaciones_programa.py`.
- Mover ahí la aceptación normal, la aceptación especial de `ÑACHEC`, la validación de duplicados, la creación de tareas con SLA y el rechazo.
- Mantener compatibilidad con imports legacy mediante wrapper.
- Agregar tests del workflow sensible.

## Resultado

- `views_derivacion_programa.py` quedó significativamente más delgada.
- El borde entre programas y `ÑACHEC` tiene ahora una capa reusable y testeable.
- El siguiente corte físico sobre esa view ya no arrastra lógica de negocio pesada.

## Riesgos residuales

- La lógica sigue dependiendo de modelos y transiciones sensibles de `ÑACHEC`.
- El siguiente bloqueo grande sigue estando en `views_institucional.py`, `views_clinico.py` y `views_nachec_*`.
