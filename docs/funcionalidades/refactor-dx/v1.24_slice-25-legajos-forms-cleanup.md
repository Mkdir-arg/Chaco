## Objetivo

Separar `legajos/forms.py` por dominio funcional para alinearlo con la modularización de views ya realizada.

## Alcance

- Extracción de forms de ciudadanía y admisión
- Extracción de forms clínicos
- Extracción de forms operativos
- Mantenimiento de compatibilidad vía `legajos/forms.py`

## Archivos

- `legajos/forms.py`
- `legajos/forms_ciudadanos.py`
- `legajos/forms_clinico.py`
- `legajos/forms_operativa.py`
- `legajos/tests/test_forms_fachada.py`

## Resultado

- El módulo histórico quedó como fachada compatible.
- La organización interna ahora sigue el mismo corte por dominio que `views_ciudadanos.py`, `views_clinico.py` y `views_operativa.py`.
- Un smoke test fija el contrato de exports del módulo de forms.

## Riesgos residuales

- El dominio `legajos` todavía mantiene otros hotspots laterales como `views_simple_contactos.py`.
- Este slice no cambió comportamiento funcional ni validaciones de negocio; fue exclusivamente una mejora estructural con compatibilidad preservada.
