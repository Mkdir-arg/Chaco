# Refactor DX Interno — Slice 17

> Fecha: 2026-03-13
> App principal: `conversaciones`
> Estado: Implementado

## User Story

Como equipo de desarrollo quiero cerrar los consumidores residuales de `conversaciones` fuera de su lista principal para reducir acoplamiento a paths fijos y estabilizar el dominio antes de pasar al siguiente hotspot.

## Alcance

- Introducir configuración global de rutas del dominio en `base.html`.
- Hacer que scripts globales de estadísticas y alertas consuman esa configuración.
- Parametrizar el path WebSocket del detalle de operador.
- Parametrizar la URL de polling de mensajes en el portal ciudadano.
- Agregar cobertura mínima del contrato renderizado del detalle.

## Cambios implementados

- `templates/includes/base.html`
  - Se agrega `window.conversacionesConfig` con rutas namespaced del dominio.
- Scripts estáticos:
  - `conversaciones_tiempo_real_global.js`
  - `conversaciones_tiempo_real.js`
  - `alertas_conversaciones_fallback.js`
  - `alertas_conversaciones_simple.js`
  - Todos dejan de embutir rutas de `conversaciones`.
- `conversaciones/templates/conversaciones/detalle.html`
  - El path WebSocket se configura desde atributo renderizado.
- `portal/templates/portal/ciudadano/consulta_detalle.html`
  - El polling de mensajes usa URL renderizada en lugar de concatenación manual.
- `conversaciones/tests/test_chat_services.py`
  - Se fija el contrato renderizado del detalle operador.

## Impacto funcional

- No hay cambio visible para usuario final.
- Se reduce deuda transversal y se prepara mejor la migración futura de rutas/namespaces.

## Riesgos residuales

- El chat ciudadano del portal sigue basado en polling HTTP.
- `portal/views_ciudadano.py` continúa como un hotspot estructural grande.
