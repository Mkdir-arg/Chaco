# Refactor DX Interno

> Version: v1.28
> Fecha: 2026-03-13
> Slice: 29
> Estado: Implementado

## Objetivo

Eliminar ambigüedad nominal en `legajos/urls.py` antes de encarar una limpieza más profunda del routing legacy.

## Alcance

- Renombre de `cerrar_alerta` del flujo de evento crítico a `cerrar_alerta_evento`
- Renombre de `cerrar_alerta` del flujo de alertas ciudadanas a `cerrar_alerta_ciudadano`
- Actualización del consumidor explícito del flujo de evento crítico
- Ampliación de smoke tests de rutas

## Archivos

- `legajos/urls.py`
- `templates/components/alertas_eventos.html`
- `core/tests/test_url_namespaces.py`

## Resultado

- `legajos/urls.py` dejó de tener `name=` duplicados para cierre de alertas.
- El contrato de rutas quedó más predecible para futuros refactors.

## Riesgos residuales

- `legajos/urls.py` sigue siendo un archivo largo y heterogéneo.
- Una reorganización mayor del routing todavía requiere revisar más consumidores y contar con mejor cobertura funcional.
