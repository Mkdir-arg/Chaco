## Descripcion

Primer slice del refactor estructural interno orientado a DX. El objetivo fue reducir lógica de negocio en views, extraer consultas reutilizables a selectors, sacar persistencia pesada de forms y dejar una base más testeable sin alterar el comportamiento funcional del sistema.

## Alcance del slice

- `users`: CRUD de usuarios del backoffice
- `portal`: registro institucional público y consulta de trámite
- `turnos`: backoffice de turnos
- `core`: helpers compartidos para geografía y mixins de UI

## Cambios principales

| Area | Cambio |
|------|--------|
| Service layer | Nuevo `users/services_admin.py`, `portal/services_registro.py`, `turnos/services_turnos.py` |
| Selectors | Nuevo `users/selectors_usuarios.py`, `portal/selectors_public.py`, `turnos/selectors_turnos.py`, `core/selectors_geografia.py` |
| Forms | `users/forms.py` dejó de persistir grupos/profile; `portal/forms_public.py` encapsula el registro institucional público |
| Views | Nuevo `users/views_admin.py`, `users/views_auth.py`, `portal/views_public.py`; `turnos/views_backoffice.py` quedó más delgado |
| Tests | Se agregaron tests del slice para usuarios, portal institucional y acciones de turnos |
