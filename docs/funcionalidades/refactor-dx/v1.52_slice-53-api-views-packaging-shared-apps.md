# Refactor DX Interno — Slice 53

> Fecha: 2026-03-13
> Estado: Implementado
> Apps afectadas: `core`, `conversaciones`

## User Story

Como equipo de desarrollo quiero mover `api_views` de las apps compartidas al mismo patrón de package sin tocar contratos HTTP ni routing.

## Alcance

- Mover `core/api_views.py` a `core/api_views/__init__.py`.
- Mover `conversaciones/api_views.py` y `conversaciones/api_extra.py` a `conversaciones/api_views/`.
- Mantener compatibilidad con imports existentes desde `api_urls.py`.
- Ampliar smoke tests de exports en `core` y `conversaciones`.

## Resultado

- Las APIs compartidas ya siguen el mismo patrón estructural que el resto del repo.
- El routing DRF/HTTP no cambió.
- La deuda estructural repo-wide quedó concentrada en APIs grandes de `legajos` o en hotspots funcionales.

## Riesgos residuales

- No se tocaron serializers, permisos ni contratos HTTP.
- La validación disponible sigue limitada a sintaxis y smoke tests de imports/exports.
