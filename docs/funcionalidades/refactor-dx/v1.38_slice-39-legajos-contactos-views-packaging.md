# Refactor DX Interno — Slice 39

> Fecha: 2026-03-13
> Estado: Implementado
> Apps afectadas: `legajos`

## User Story

Como equipo de desarrollo quiero agrupar en `legajos/views/` las views auxiliares de contactos y dashboards simples antes de mover bloques más sensibles.

## Alcance

- Crear `legajos/views/` como paquete real para el subdominio de contactos auxiliares.
- Mover al paquete las APIs, paneles y dashboards simples de contactos.
- Mantener compatibilidad con imports y URLs legacy mediante wrappers.
- Ampliar tests de fachada para validar exports del paquete.

## Resultado

- La migración de `views` en `legajos` ya no está bloqueada en todo o nada.
- Los módulos legacy siguen funcionando.
- El proyecto ya puede seguir por subdominios concretos sin forzar un packaging masivo de toda la app.

## Riesgos residuales

- Clínica, institucional y `ÑACHEC` siguen fuera del paquete nuevo.
- La siguiente etapa debe seguir por subdominio, no por reemplazo total de todas las views de `legajos`.
