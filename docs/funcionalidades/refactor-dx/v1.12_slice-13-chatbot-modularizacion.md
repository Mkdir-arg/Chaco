# Refactor DX Interno — Slice 13: `chatbot` modularización

> Version: v1.12  
> Fecha: 2026-03-13  
> Estado: Implementado

## Objetivo

Separar la superficie pública y administrativa de `chatbot`, validando payloads JSON y extrayendo lecturas/workflows reutilizables sin cambiar la superficie HTTP.

## Alcance

- `chatbot/views.py` como fachada compatible
- Nuevos módulos:
  - `forms_chatbot.py`
  - `selectors_chatbot.py`
  - `services_chatbot.py`
  - `views_public.py`
  - `views_admin.py`
- Tests base del slice

## Cambios implementados

- El chat del usuario quedó separado del panel admin
- Los payloads JSON dejaron de validarse inline
- El dashboard admin y las conversaciones del usuario ahora usan selectors explícitos
- El workflow principal del bot y las operaciones de feedback/conocimiento quedaron centralizadas en services

## Impacto en DX

- Baja el tamaño cognitivo de `chatbot/views.py`
- Facilita testear y modificar la parte pública sin tocar administración
- Alinea el módulo con el patrón que ya usan `conversaciones`, `legajos` y `configuracion`

## Riesgos residuales

- Se mantuvieron `@csrf_exempt` legacy en endpoints JSON para no romper el frontend actual
- El hardening de seguridad del bot queda pendiente de un corte específico

## Siguiente etapa sugerida

- Definir un corte de hardening para endpoints JSON (`chatbot` y `conversaciones`)
- O cerrar la migración de consumidores sensibles de URLs en autenticación
