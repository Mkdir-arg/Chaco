# Refactor DX Interno — Slice 48

> Fecha: 2026-03-13
> Estado: Implementado
> Apps afectadas: `legajos`, `conversaciones`

## User Story

Como equipo de desarrollo quiero cerrar el packaging restante de `services` en `legajos` para que el wiring posterior de `signals` dependa de una API estable y consistente.

## Alcance

- Mover `services_alertas.py`, `services_filtros_usuario.py`, `services_institucional.py` y `services_nachec.py` a `legajos/services/`.
- Mantener compatibilidad con imports legacy mediante wrappers mínimos.
- Migrar consumidores internos claros al paquete `legajos.services`.
- Ampliar smoke tests del paquete de servicios.

## Resultado

- `legajos/services/` ya concentra también las familias residuales de alertas, filtros de usuario, institucional y `ÑACHEC`.
- Los módulos legacy siguen funcionando como wrappers y se redujo el acoplamiento directo a archivos planos.
- `conversaciones` y señales auxiliares ya consumen la API de paquete nueva.

## Riesgos residuales

- El wiring de `signals` en `legajos/apps.py` sigue siendo el siguiente frente sensible.
- No hubo cambio funcional deliberado; la validación disponible sigue limitada a sintaxis y smoke tests de exports.
