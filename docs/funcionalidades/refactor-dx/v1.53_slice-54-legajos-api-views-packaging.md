## User Story

Como equipo de desarrollo quiero cerrar también `api_views` de `legajos` para agotar la deuda estructural repo-wide más obvia sin tocar contratos DRF.

## Alcance

- Mover `legajos/api_views.py` a `legajos/api_views/__init__.py`.
- Mover `legajos/api_views_contactos.py` a `legajos/api_views/contactos.py`.
- Mantener compatibilidad con imports existentes desde `api_urls.py` y `api_urls_contactos.py`.
- Ampliar smoke tests de exports del paquete.

## Resultado

- `legajos` ya no deja la API DRF fuera de la convención de paquetes.
- El routing de la API se mantuvo estable.
- La deuda estructural repo-wide quedó prácticamente agotada.

## Riesgos residuales

- No se tocaron serializers, permisos ni contratos DRF.
- La validación disponible sigue limitada a sintaxis y smoke tests de imports/exports.
