# Refactor DX Interno

> Version: v1.26
> Fecha: 2026-03-13
> Slice: 27
> Estado: Implementado

## Objetivo

Desacoplar el módulo legacy de contactos separando panel, APIs, lectura compuesta y uploads de adjuntos.

## Alcance

- Extracción de queries/payloads a `selectors_contactos.py`
- Extracción del manejo de adjuntos a `services_contactos.py`
- Separación de vistas HTML y APIs en módulos distintos
- Conservación de compatibilidad vía `views_simple_contactos.py`

## Archivos

- `legajos/selectors_contactos.py`
- `legajos/services_contactos.py`
- `legajos/views_contactos_panel.py`
- `legajos/views_contactos_api.py`
- `legajos/views_simple_contactos.py`
- `legajos/tests/test_contactos_fachada.py`

## Resultado

- El módulo legacy quedó como fachada compatible.
- Las consultas de actividades, timeline, evolución y archivos ya no viven inline en views.
- El upload/delete de adjuntos tiene una única fuente de validación.

## Correcciones incluidas

- Uso de `EventoCritico.detalle` en lugar de un campo legacy inexistente.
- Uso de `PlanIntervencion.actividades` en lugar de referencias legacy a `objetivos`.

## Riesgos residuales

- El módulo todavía conserva permisos legacy en varias APIs; este slice no endureció seguridad para evitar cambios funcionales amplios.
- Sigue pendiente revisar si parte de estas APIs conviene mover a endpoints más explícitos o namespaced en una etapa posterior.
