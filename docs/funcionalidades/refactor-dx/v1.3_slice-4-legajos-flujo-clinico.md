## Descripcion

Cuarto slice del refactor estructural interno orientado a DX. El foco estuvo en el flujo clínico base del `legajo de atención`: evaluación inicial, plan de intervención, seguimientos, derivaciones y acciones de cierre/reapertura. El objetivo fue sacar orquestación y queries repetidas de las views, ordenar forms y corregir inconsistencias reales del template del plan.

## Alcance del slice

- `legajos`: listado de legajos
- `legajos`: detalle del legajo
- `legajos`: evaluación inicial
- `legajos`: planes de intervención
- `legajos`: seguimientos
- `legajos`: derivaciones
- `legajos`: cierre y reapertura de legajo

## Cambios principales

| Area | Cambio |
|------|--------|
| Service layer | Nuevo `legajos/services_legajos.py` para evaluación, planes, seguimientos, derivaciones y cambios de estado |
| Selectors | Nuevo `legajos/selectors_legajos.py` para listados, detalle y métricas de seguimientos |
| Forms | `EvaluacionInicialForm` y `PlanIntervencionForm` ahora se enfocan en mapeo/validación; nuevo `LegajoReabrirForm` |
| Views | `legajos/views.py` delega queries y side effects del flujo clínico base |
| Templates | `plan_form.html` deja de ignorar valores existentes y preserva actividades dinámicas; cierre/reapertura pasan a renderizar forms Django |
| Tests | Se agregan tests del slice para evaluación, plan dinámico, seguimientos, métricas y reapertura |

## Notas técnicas

- La edición de planes ya no depende de inputs hardcodeados vacíos.
- El parseo de actividades dinámicas se preservó para no cortar planes legacy con más de tres slots enviados por `POST`.
- La ejecución de tests Django sigue bloqueada en este entorno por configuración faltante (`DJANGO_SECRET_KEY`) y dependencia de runtime ausente (`channels`); la sintaxis del slice se validó con `py_compile`.
