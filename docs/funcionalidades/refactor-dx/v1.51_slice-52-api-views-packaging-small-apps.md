# Refactor DX Interno — Slice 52

> Fecha: 2026-03-13
> Estado: Implementado
> Apps afectadas: `dashboard`, `users`, `chatbot`

## User Story

Como equipo de desarrollo quiero mover `api_views` de las apps chicas al mismo patrón de package para no dejar excepciones entre views HTML y API.

## Alcance

- Mover `dashboard/api_views.py`, `users/api_views.py` y `chatbot/api_views.py` a paquetes reales.
- Mantener compatibilidad con imports existentes desde `urls.py` y `api_urls.py`.
- Ampliar smoke tests de exports en esas apps.

## Resultado

- Las APIs chicas ya no quedan fuera de la convención repo-wide.
- Los import paths desde routing no cambiaron.
- La deuda estructural restante quedó concentrada en APIs más sensibles o voluminosas.

## Riesgos residuales

- No se tocaron contratos HTTP ni serializers.
- La validación disponible sigue limitada a sintaxis y smoke tests de imports/exports.
