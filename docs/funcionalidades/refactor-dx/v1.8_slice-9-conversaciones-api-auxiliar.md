# Refactor DX Interno — Slice 9: `conversaciones` API auxiliar

> Version: v1.8  
> Fecha: 2026-03-13  
> Estado: Implementado

## Objetivo

Eliminar la divergencia arquitectónica entre la UI principal de `conversaciones` y sus APIs auxiliares de alertas/detalle en vivo.

## Alcance

- Refactor de:
  - `conversaciones/api_views.py`
  - `conversaciones/api_extra.py`
- Reutilización de:
  - `selectors_conversaciones.py`
  - `services_chat.py`
- Ampliación de tests existentes

## Cambios implementados

- Los permisos del módulo ahora se resuelven con una misma función reutilizable
- Las alertas y previews del dropdown consumen selectors explícitos
- El marcado de mensajes leídos pasó a un service compartido
- El detalle mínimo en vivo reutiliza el selector anotado del módulo en lugar de repetir queries

## Impacto en DX

- Reduce duplicación dentro de `conversaciones`
- Evita divergencias entre endpoints HTML y endpoints auxiliares
- Deja más claro dónde viven las lecturas y dónde viven los side effects

## Riesgos residuales

- La deuda de `@csrf_exempt` y del transporte realtime/polling sigue abierta
- Las APIs del chat siguen siendo un mix de DRF liviano y `JsonResponse` legacy

## Siguiente etapa sugerida

- Definir un corte específico de hardening para `conversaciones`
- Revisar si conviene unificar gradualmente la estrategia de respuestas JSON del módulo
