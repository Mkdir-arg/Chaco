# Refactor DX Interno — Slice 22: `ÑACHEC` prestaciones, cierre y dashboard

> Versión: v1.21
> Fecha: 2026-03-13
> Estado: Implementado

## Objetivo

Reducir el mayor hotspot del repo en `legajos/views_nachec.py` sin tocar todavía los flujos más sensibles de validación, relevamiento y evaluación.

## Cambios aplicados

- Se creó `legajos/views_nachec_prestaciones.py` para gestión de prestaciones.
- Se creó `legajos/views_nachec_cierre.py` para cierre y reapertura de casos.
- Se creó `legajos/views_nachec_dashboard.py` para el dashboard del programa.
- `legajos/views_nachec.py` quedó como fachada compatible para esos exports.
- Se agregó `legajos/tests/test_nachec_fachada.py` para fijar el contrato de exports desde la fachada.

## Impacto funcional

- No hay cambios de modelo ni migraciones.
- No cambian URLs ni templates.
- `urls_nachec.py` sigue importando desde `views_nachec.py`, pero las implementaciones reales ya viven en módulos dedicados.

## Beneficio en DX

- `legajos/views_nachec.py` baja de 2712 líneas a 1618 líneas.
- El módulo deja de mezclar prestaciones, cierre y dashboard con el resto del workflow.
- La próxima etapa sobre `ÑACHEC` puede enfocarse en relevamiento/evaluación con menor costo cognitivo.

## Riesgos residuales

- Siguen dentro del monolito los bloques de validación, asignación, relevamiento, evaluación y activación de plan.
- La lógica interna aún no está separada en selectors/services; este slice solo resolvió la fragmentación física inicial del módulo.
