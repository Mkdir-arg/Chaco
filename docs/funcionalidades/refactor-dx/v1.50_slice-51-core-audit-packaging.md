# Refactor DX Interno — Slice 51

> Fecha: 2026-03-13
> Estado: Implementado
> Apps afectadas: `core`

## User Story

Como equipo de desarrollo quiero cerrar la deuda estructural pendiente de `core` agrupando auditoría, performance y señales dentro de paquetes reales.

## Alcance

- Mover `views_auditoria.py` y `performance_dashboard.py` al paquete `core/views/`.
- Crear `core/signals/` para auditoría, historial y cache.
- Mantener wrappers compatibles en los módulos históricos.
- Actualizar `CoreConfig.ready()` y URLs para consumir el paquete nuevo.
- Ampliar smoke tests de exports del paquete.

## Resultado

- `core` ya no deja fuera de la convención sus bloques de auditoría/performance.
- El wiring de señales quedó explícito desde `AppConfig`.
- Los imports legacy siguen funcionando mediante wrappers delgados.

## Riesgos residuales

- No se modificó la lógica de auditoría ni de performance.
- La validación disponible sigue limitada a sintaxis y smoke tests de imports/exports.
