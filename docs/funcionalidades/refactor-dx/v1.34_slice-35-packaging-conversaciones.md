## Objetivo

Empaquetar `conversaciones` bajo carpetas reales de responsabilidad sin romper el chat, el runtime de backoffice ni el wiring de señales.

- `conversaciones.views` convertido en paquete real
- Nuevo paquete `conversaciones.forms`
- Nuevo paquete `conversaciones.services`
- Nuevo paquete `conversaciones.selectors`
- Nuevo paquete `conversaciones.signals`
- Wrappers de compatibilidad para módulos legacy
- Unificación de `AppConfig.ready()`
- Actualización de tests de patching al path real del paquete
- Smoke test de exports públicos

## Resultado

- `conversaciones` quedó alineada con el patrón de packaging del refactor.
- El wiring de señales dejó de depender de una clase `AppConfig` inconsistente.
- Los tests del módulo ya apuntan a los paths reales cuando hacen patch de internals.

## Riesgos residuales

- Sigue faltando validación funcional completa con Channels en runtime real.
- `api_views.py`, `api_extra.py`, `consumers.py` y `routing.py` todavía no usan un packaging más profundo, aunque ya quedaron apoyados sobre paquetes internos estables.
