## Descripcion

Tercer slice del refactor estructural interno orientado a DX. El foco estuvo en el punto de entrada de `legajos`: listado y detalle de ciudadanos, consulta RENAPER y wizard de admisión. El objetivo fue sacar queries y manejo de sesión de las views, corregir inconsistencias de forms y limpiar deuda directa en URLs duplicadas.

## Alcance del slice

- `legajos`: listado de ciudadanos
- `legajos`: detalle de ciudadano
- `legajos`: consulta RENAPER para alta
- `legajos`: alta manual de ciudadano
- `legajos`: confirmación de alta con datos de RENAPER
- `legajos`: wizard de admisión en 3 pasos

## Cambios principales

| Area | Cambio |
|------|--------|
| Service layer | Nuevo `legajos/services_ciudadanos.py` para RENAPER/cache y `legajos/services_admision.py` para sesión del wizard |
| Selectors | Nuevo `legajos/selectors_ciudadanos.py` para listado, métricas y detalle del ciudadano |
| Forms | Nuevos `CiudadanoManualForm`, `CiudadanoConfirmarForm`, `CiudadanoUpdateForm` para separar contextos |
| Views | `legajos/views.py` delega detalle/listado, RENAPER y admisión en services/selectors |
| URLs | `legajos/urls.py` elimina duplicación real de rutas base |
| Templates | `ciudadano_renaper_form.html` conserva valores ingresados y corrige JS/choices inconsistentes |
| Tests | Se agregan tests del slice para forms por contexto y manejo de sesión en admisión |
