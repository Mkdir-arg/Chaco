# Refactor DX Interno — Slice 14: `chatbot` contrato y CSRF

> Version: v1.13  
> Fecha: 2026-03-13  
> Estado: Implementado

## Objetivo

Corregir la divergencia real entre el frontend y el backend del módulo `chatbot` y retirar dependencias innecesarias de `@csrf_exempt`.

## Alcance

- Alineación de rutas consumidas por `chat.js`
- Alineación del shape JSON de `send_message`
- URLs renderizadas desde template para evitar hardcodes
- Retiro de `@csrf_exempt` en endpoints principales
- Tests del contrato y de CSRF

## Cambios implementados

- `chat_interface.html` y `admin_dashboard.html` ahora exponen URLs concretas al JS vía `data-*`
- `chat.js` dejó de depender de rutas hardcodeadas inconsistentes
- `views_public.send_message()` ahora devuelve `conversation_id`, `user_message` y `assistant_message`
- `views_public.py` y `views_admin.py` ya no usan `@csrf_exempt` en los endpoints principales del módulo

## Impacto en DX

- Reduce bugs silenciosos entre frontend y backend
- Hace explícito el contrato HTTP/JSON del módulo
- Deja el bot en una posición más segura y mantenible

## Riesgos residuales

- `chatbot/api_views.py` sigue siendo una superficie separada que podría revisarse si se amplía la API del bot
- La deuda de hardening grande sigue concentrada en `conversaciones`, no ya en `chatbot`

## Siguiente etapa sugerida

- Atacar `conversaciones` con el mismo enfoque: alinear contrato frontend/backend y retirar `@csrf_exempt`
- O cerrar la migración sensible de URLs en autenticación
