# Configuración del Sistema — v1.1 ABM Secretarías y Subsecretarías

> Version: v1.1
> Fecha: 2026-03-15
> Tipo: EVOLUTIVO
> App/Modulo: `core`, `configuracion`, `legajos`

---

## Descripcion

Agrega la jerarquía organizacional estatal al sistema: Secretaría → Subsecretaría → Programa. Permite a usuarios con rol `secretariaConfigurar` gestionar el catálogo de secretarías y subsecretarías, y vincular programas a una subsecretaría para clasificarlos dentro de la estructura del organismo. Es el Paso 1 de 4 del roadmap de configuración de programas (precede al wizard US-005).

## User Story

```
Como usuario con rol `secretariaConfigurar` quiero gestionar el catálogo de Secretarías y Subsecretarías y vincular Programas a una Subsecretaría para organizar los programas dentro de la jerarquía organizacional
```

## Criterios de aceptacion

- [x] CRUD completo de Secretarías (nombre único, descripción, estado activo/inactivo)
- [x] CRUD completo de Subsecretarías (nombre único por secretaría, FK a Secretaría)
- [x] Al eliminar una Secretaría con subsecretarías → error amigable (no 500)
- [x] Al eliminar una Subsecretaría con programas → error amigable (no 500)
- [x] Confirmación de eliminación con SweetAlert2 en las listas
- [x] Filtro de subsecretarías por secretaría en el listado
- [x] Campo `subsecretaria` nullable en `Programa` para no romper datos existentes
- [x] Acceso restringido a rol `secretariaConfigurar`

- Usada por: US-005 (wizard de configuración de programas — vincula al crear/editar programa)

## Decisiones tecnicas

- **`on_delete=PROTECT` en todas las FKs**: las eliminaciones en cascada son peligrosas en datos organizacionales. Si hay dependientes, el sistema muestra un mensaje amigable en lugar de un 500.
- **FK `subsecretaria` nullable en `Programa`**: los programas existentes no tienen subsecretaría asignada. Se fuerza la asignación desde US-005 (wizard), no retroactivamente.
- **`unique_together = [('nombre', 'secretaria')]`** en Subsecretaría: permite dos subsecretarías con el mismo nombre en distintas secretarías (ej. "Administración" puede existir en múltiples secretarías).
- **Cross-app FK como string**: `'core.Subsecretaria'` en `legajos/models_programas.py` para evitar importación circular.
- **Migración manual**: Docker no estaba corriendo durante la implementación. Las migraciones se escribieron manualmente siguiendo la numeración correcta (core:0008, legajos:0024).
- **`puede_eliminarse()`**: método auxiliar en el model para verificar si tiene dependientes antes de intentar eliminar. Las DeleteViews capturan `ProtectedError` de todas formas, pero el método queda disponible para futuros checks de UI.

## Limitaciones conocidas / deuda tecnica

- La pantalla de edición de Programa no muestra aún el campo `subsecretaria` — queda pendiente para US-005 (wizard de programa).
- No hay validación de que la Secretaría esté activa al asignar una Subsecretaría activa. Podría agregarse en el form en el futuro.

## Cambios respecto a version anterior

La v1.0 de Configuración del Sistema solo incluía la gestión de geografía e instituciones. Esta versión agrega la jerarquía organizacional como entidades de primer nivel en `core`, accesibles desde `configuracion`.
