## Descripcion

Módulo de administración de tablas maestras y configuración general del sistema. Permite gestionar la división geográfica (provincias, municipios, localidades), las instituciones de la red y sus componentes (personal, evaluaciones, planes de fortalecimiento, indicadores), actividades institucionales con control de asistencia y derivaciones institucionales. Actúa como backend de configuración para múltiples módulos.

## User Story

```
Como administrador del sistema quiero gestionar las tablas maestras de geografía e instituciones para que el resto del sistema pueda referenciarlas correctamente y los operadores puedan seleccionar valores válidos en los formularios.
```

## Modelos principales

Los modelos de configuración están en `core/models.py` y `legajos/models.py`. La app `configuracion` no tiene modelos propios (el archivo está vacío), solo vistas y formularios que gestionan esos modelos.

| Modelo gestionado | App origen | Descripcion |
|--------|-------------|-------------|
| `Provincia` | `core` | Provincias del país |
| `Municipio` | `core` | Municipios por provincia |
| `Localidad` | `core` | Localidades por municipio |
| `Institucion` | `core` | Instituciones de la red (CRUD completo) |
| `LegajoInstitucional` | `legajos` | Legajo de seguimiento institucional |
| `PersonalInstitucion` | `legajos` | Personal de cada institución |
| `EvaluacionInstitucional` | `legajos` | Evaluaciones periódicas |
| `PlanFortalecimiento` | `legajos` | Planes de mejora institucional |
| `IndicadorInstitucional` | `legajos` | Indicadores de monitoreo |
| `StaffActividad` | `legajos` | Staff asignado a actividades institucionales |

## Integraciones con otras funcionalidades

- Instituciones: este módulo maneja la parte de configuración y seguimiento institucional
- Gestión de Ciudadanos: `Provincia`, `Municipio`, `Localidad` son usados como FK en `Ciudadano`
- Programas Sociales: `PlanFortalecimiento` puede tener `ConfiguracionTurnos`
- Portal Ciudadano: la API de municipios y localidades (`/portal/api/municipios/`, `/portal/api/localidades/`) sirve datos geográficos para formularios dinámicos

## Notas técnicas

- La app `configuracion` no tiene modelos propios; actúa como capa de UI sobre modelos de `core` y `legajos`.
- Las rutas `/configuracion/dispositivos/` son un alias legacy de instituciones (compatibilidad).
- `DispositivoRed` y `DispositivoForm` en `configuracion/views.py` son aliases de `Institucion` e `InstitucionForm`.
- El formulario de provincia hace un redirect con timestamp (`?t=...`) para invalidar caché de listado.
