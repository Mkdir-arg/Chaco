## Descripcion

El Legajo de Atención es el expediente clínico-social de un ciudadano dentro del sistema SEDRONAR. Centraliza la evaluación inicial, el plan de intervención, los seguimientos, las derivaciones, los eventos críticos y los consentimientos informados. Es la unidad de trabajo principal de los profesionales de atención.

## User Story

```
Como profesional de atención quiero gestionar el legajo de un ciudadano para registrar su evaluación inicial, armar un plan de intervención, hacer seguimientos periódicos y derivarlo cuando sea necesario.
```

## Modelos principales

| Modelo | Descripcion | Campos clave |
|--------|-------------|-------------|
| `LegajoAtencion` | Legajo principal (hereda `LegajoBase`) | `ciudadano`, `estado` (ABIERTO/EN_SEGUIMIENTO/DERIVADO/CERRADO), `responsable`, `confidencialidad`, `codigo` (UUID) |

| `PlanIntervencion` | Plan de trabajo con etapas | `legajo`, `objetivos`, `etapas` (JSONField), `vigente` |
| `SeguimientoContacto` | Registro de cada contacto de seguimiento | `legajo`, `tipo`, `fecha`, `resultado`, `profesional` |
| `Derivacion` | Derivación a institución de la red | `legajo`, `institucion_destino`, `estado`, `urgencia`, `motivo` |
| `EventoCritico` | Evento de alta prioridad (OD, recaída, etc.) | `legajo`, `tipo`, `descripcion`, `fecha`, `resuelto` |
| `AlertaEventoCritico` | Alerta generada automáticamente por evento | `evento`, `destinatarios`, `leida` |

## Integraciones con otras funcionalidades

- Gestión de Ciudadanos: cada legajo pertenece a un ciudadano
- Instituciones: las derivaciones apuntan a instituciones de la red SEDRONAR
- Programas Sociales: los legajos se pueden inscribir en programas vía solapas dinámicas
- Alertas: los eventos críticos generan alertas automáticas
- Contactos y Red: historial de contactos y vínculos familiares vinculados al legajo
- Auditoría: cada cambio en datos clínicos sensibles genera registros en `AuditoriaEvaluacion`

## Notas técnicas

- El ID del legajo es UUID para evitar enumeración.
- El campo `estado` tiene 4 valores: ABIERTO, EN_SEGUIMIENTO, DERIVADO, CERRADO.
- La `EvaluacionInicial` tiene flags críticos (`riesgo_suicida`, `violencia`) que disparan auditoría especial y alertas.
- Las etapas del plan de intervención se almacenan en un `JSONField`.
- El modelo `LegajoBase` es abstracto y define `codigo`, `estado`, `responsable`, `confidencialidad` y `notas`.
