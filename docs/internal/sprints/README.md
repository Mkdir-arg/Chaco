# Sprints

Cada archivo documenta un sprint completo: qué se planificó, qué se analizó, qué se definió y el estado de cada requerimiento.

## Convención de nombres

`YYYY-MM-DD-sprint-NNN.md` — fecha de inicio del sprint + número secuencial.

## Registro

## Estados posibles

- **Activo** — sprint en curso
- **Cerrado** — sprint finalizado con review
- **Cancelado** — sprint interrumpido
