# Decisiones técnicas (ADRs)

Registro de decisiones de arquitectura relevantes para el proyecto.
Cada decisión tiene su propio archivo con contexto, opciones evaluadas y consecuencias.

## Formato de nombre

`NNN-titulo-en-kebab-case.md` — donde NNN es número secuencial con ceros (`001`, `002`, ...).

## Registro

| ADR | Título | Estado |
|---|---|---|
| [001](001-stack-principal.md) | Stack principal del sistema | Aceptado |
| [002](002-patron-selectors-services.md) | Patrón selectors/services en apps Django | Aceptado |

## Estados posibles

- **Propuesto** — en discusión
- **Aceptado** — decisión tomada e implementada
- **Deprecado** — reemplazado por otra decisión
- **Rechazado** — evaluado y descartado
